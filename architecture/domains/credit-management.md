# Domain cutover plan: credit-management

**Cutover order: 6 of 6** (LAST — money-equivalent state.)

## Surface today

- Backend gateway: `supabase/functions/_shared/modules/credit-management/gateway.ts`
- Edge endpoint: `supabase/functions/usage-credits/index.ts` → `getMyBalance`
- Frontend gateway: `src/modules/credit-management/gateway.ts`
- Tables read (RLS-enforced): `core_user_profiles.credits_balance`,
  `billing_credit_transactions`
- Tables written: `billing_credit_transactions` (today: not written from edge;
  future: via service-role write from worker on job completion / refund).
  `core_user_profiles.credits_balance` is **guarded by trigger**
  (`guard_profile_updates`) — only admins or service role can change it.

## Why LAST

- Any divergence is real revenue/credit drift, not just bad UX.
- We will only attempt this after every other domain has been cut over and
  rolled back at least once successfully in staging — so the freeze and
  reconciliation tooling is fully proven.

## Phases

| Phase         | Action                                                                                  |
|---------------|-----------------------------------------------------------------------------------------|
| `active`      | Current state. Default.                                                                  |
| `dual-write`  | When credit writes exist: write `billing_credit_transactions` to old + new with same id. |
| `shadow-read` | Compare balance returned by old vs new for same user. Discrepancy → halt.                |
| `cutover`     | Frontend reads balance from new. Worker writes credits to new only.                      |
| `frozen`      | Both surfaces refuse credit writes for ~60s. NO new jobs accepted in this window.        |
| `rolled-back` | New refuses writes; old resumes.                                                         |

## Freeze-write plan

Coordinated with `job-orchestrator`:
1. `CUTOVER_JOB_ORCHESTRATOR=frozen` first (no new debits incoming).
2. `CUTOVER_CREDIT_MANAGEMENT=frozen` on both surfaces.
3. Verify in-flight transactions drained:
   ```sql
   SELECT count(*) FROM billing_credit_transactions
   WHERE created_at > now() - interval '5 minutes';
   ```
   Should be stable across two consecutive checks 30s apart.
4. Run balance-equality check between old and new (see Reconciliation).
5. If equal: flip new to `cutover`, then job-orchestrator back to `cutover`.
6. If not equal: abort, revert to `active` on both, escalate.

## Dual-write plan

- Window: as short as possible — target under 30 minutes.
- Use the SAME transaction `id` (UUID generated by writer) on both surfaces.
- If new insert fails after old insert succeeded: alert immediately, leave
  old as canonical, do NOT retry on new without operator approval.
- Reads stay on old until `cutover`.

## Rollback

```bash
CUTOVER_CREDIT_MANAGEMENT=rolled-back
VITE_CUTOVER_CREDIT_MANAGEMENT=rolled-back
# Also re-freeze jobs while reconciling, if rollback happens during cutover:
CUTOVER_JOB_ORCHESTRATOR=frozen
```

Smoke test:
```bash
curl -fsS -H "Authorization: Bearer $TOKEN" \
  https://<project-ref>.supabase.co/functions/v1/usage-credits | jq '.credits_balance'
```

Reconciliation (must hold AT ALL TIMES, especially before flipping cutover):
```sql
-- Balance derived from transactions must equal stored balance
SELECT p.id,
       p.credits_balance AS stored,
       COALESCE(SUM(t.amount), 0) AS derived
FROM core_user_profiles p
LEFT JOIN billing_credit_transactions t ON t.user_id = p.id
GROUP BY p.id, p.credits_balance
HAVING p.credits_balance <> COALESCE(SUM(t.amount), 0);
-- Must return 0 rows. Any row = halt cutover.
```

## RLS notes

Existing policies preserved exactly:
- `profiles: users select own` / `profiles: admins select all` /
  `profiles: users update own` / `profiles: admins update all`
- `guard_profile_updates` trigger keeps `credits_balance` writable only by
  admin or service role.
- `credits: users select own` / `credits: admins select all`

Cutover never modifies any of these. The trigger is the safety net that
prevents a cutover bug from letting a user mint credits.
