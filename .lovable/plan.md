## مشکل

پیش‌نمایش ویدیو در نسبت‌های مختلف (به‌خصوص 9:16 و 1:1) همچنان زیر باکس چت می‌رود. علت: کف ارتفاع `PREVIEW_MAX_HEIGHT = 'calc(100vh - 17rem)'` ثابت است و ارتفاع واقعی کامپوزر — که با تعداد ردیف‌های textarea، خطای خطا، و متن راهنمای زیر آن تغییر می‌کند — می‌تواند بیشتر از 17rem شود. علاوه بر این، عنصر `<main>` با `min-h-screen + place-items-center` کارت پیش‌نمایش را در مرکز کل ارتفاع viewport قرار می‌دهد و `pb-40` هنگام مرکزچینی نادیده گرفته می‌شود.

## راه‌حل: اندازه‌گیری زنده‌ی کامپوزر

به‌جای ثابت کردن ارتفاع، موقعیت بالای کامپوزر را با `ResizeObserver` اندازه می‌گیریم و فاصله‌ی آن تا بالای viewport را به‌عنوان سقف پیش‌نمایش استفاده می‌کنیم. این تضمین می‌کند که در هر اندازه‌ی صفحه، هر نسبت تصویری، و هر تعداد خط متن در کامپوزر، پیش‌نمایش کاملاً بالای چت باکس بماند.

### تغییرات در `src/modules/generator-ui/pages/DashboardPage.tsx`

1. **افزودن state و ref برای اندازه‌گیری کامپوزر:**
   ```tsx
   const composerRef = useRef<HTMLFormElement | null>(null)
   const [previewMaxHeightPx, setPreviewMaxHeightPx] = useState<number>(
     () => Math.max(240, (typeof window !== 'undefined' ? window.innerHeight : 900) - 320)
   )
   ```

2. **افزودن `useEffect` با `ResizeObserver`:** بالای کامپوزر و ارتفاع viewport را رصد می‌کند، یک حاشیه‌ی امن 24px لحاظ می‌کند، و `previewMaxHeightPx` را به‌روز می‌کند. روی window resize نیز recompute می‌شود.

3. **اتصال ref به `<form>` کامپوزر** (خط 2316-2319).

4. **بازنویسی `ratioToHeight` و `ratioToWidth`** برای استفاده از `${previewMaxHeightPx}px` به‌جای `calc(100vh - 17rem)` ثابت.

5. **اصلاح `<main>` (خط 1778):** تغییر از `min-h-screen ... pb-40` به ارتفاع‌محدود مبتنی بر `previewMaxHeightPx` با ارتفاع inline (top header ~3rem) تا مرکزچینی واقعاً در فضای امن انجام شود؛ همچنین حذف `-translate-y-6` که سبب لغزش بصری می‌شد.

6. **افزودن `maxHeight: previewMaxHeightPx` به کارت پیش‌نمایش** به‌عنوان لایه‌ی دفاعی دوم.

### چرا کار می‌کند

- ارتفاع کامپوزر بالا و پایین می‌رود (پر کردن متن، نمایش خطا، باز شدن گزینه‌ها) — `ResizeObserver` بلافاصله مقدار جدید را گزارش می‌دهد.
- روی viewportهای کوتاه و بلند یکسان درست کار می‌کند چون مقدار به جای واحدهای ثابت، از top کامپوزر گرفته می‌شود.
- نسبت‌های 9:16، 1:1 و 16:9 همگی از همین سقف پیکسلی پیروی می‌کنند، پس هیچ‌کدام زیر چت‌باکس نمی‌روند.
