## هدف نهایی
دکمه‌ی دانلود فیلم نهایی همیشه یک فایل با فرمت استاندارد و سازگار با همه‌جا (MP4 / H.264 + AAC) تحویل بدهد، نه WebM.

## وضعیت فعلی
- خروجی Final Film از `MediaRecorder` ساخته می‌شود و معمولاً WebM است (`mergeRes.extension === 'webm'`).
- هنگام دانلود (`DashboardPage.tsx`، کارت‌های Library و کارت‌های Storage/Archive)، همان فایل ذخیره‌شده با پسوند واقعی‌اش (`.webm`) دانلود می‌شود.
- WebM در QuickTime، Windows Media Player، گالری موبایل و بعضی ابزارها پخش نمی‌شود.
- یک ابزار آماده و کامل به نام `ensureMp4` در `src/modules/generator-ui/lib/transcodeToMp4.ts` وجود دارد که در مرورگر با ffmpeg.wasm فایل را به MP4 استاندارد (H.264 + AAC + faststart) تبدیل می‌کند، اما در مسیر دانلود استفاده نمی‌شود.

## برنامه اصلاح
1. **تبدیل به MP4 هنگام دانلود (کارت‌های Library — Final Videos)**
   - وقتی کاربر دکمه دانلود را می‌زند: فایل با URL پروکسی‌شده fetch می‌شود، سپس از `ensureMp4` عبور می‌کند و خروجی با نام `final-film-xxxx.mp4` ذخیره می‌شود.
   - اگر فایل از قبل MP4 سالم باشد، `ensureMp4` فقط remux سریع (faststart) انجام می‌دهد.
   - حین تبدیل، روی همان دکمه یک حالت loading (اسپینر) نمایش داده می‌شود تا کاربر بداند در حال آماده‌سازی است.

2. **همین رفتار برای کارت‌های Storage / Archive**
   - همان مسیر دانلود → `ensureMp4` → خروجی `.mp4` اعمال می‌شود تا همه‌ی نقاط دانلود یک خروجی استاندارد بدهند (یک مسیر واحد).

3. **مدیریت خطا و فایل‌های بزرگ (رفتار امن)**
   - `ensureMp4` برای فایل‌های خیلی بزرگ (بالای سقف داخلی) خطا می‌دهد یا ممکن است شکست بخورد. در این حالت به‌صورت fallback همان فایل اصلی با پسوند واقعی‌اش دانلود می‌شود تا کاربر هیچ‌وقت بدون خروجی نماند.
   - پیام/لاگ خطا بدون شکستن UI مدیریت می‌شود.

4. **اعتبارسنجی**
   - بررسی می‌شود دانلود از Library و Storage یک فایل `.mp4` قابل پخش می‌دهد.
   - مطمئن می‌شویم تبدیل، فقط در لحظه‌ی دانلود اجرا می‌شود و منطق ساخت/ذخیره Final Film و آرشیو تغییری نمی‌کند.

## فایل‌های مورد تغییر
- `src/modules/generator-ui/pages/DashboardPage.tsx` (هندلرهای دانلود Library و Storage + state اسپینر دانلود)
- بدون تغییر منطقی در `transcodeToMp4.ts` (فقط استفاده می‌شود)

## جزئیات فنی
- از تابع موجود `ensureMp4(blob, mimeType)` استفاده می‌شود؛ نیازی به وابستگی جدید نیست (ffmpeg.wasm از قبل در پروژه هست).
- یک state مثل `downloadingId: string | null` برای نشان دادن اسپینر روی دکمه‌ی همان کارت اضافه می‌شود.
- در صورت خطای transcode، fallback به دانلود فایل اصلی (`webm/mp4`) حفظ می‌شود.
