مشکل دقیق: تابع `image-reframe` الان فقط با prompt و یک تصویر مرجعِ نسبت تصویر تلاش می‌کند Nano Banana را مجبور به خروجی 9:16 کند. لاگ تابع نشان می‌دهد مدل در هر دو تلاش همان نسبت منبع را برگردانده: `1248x832` یعنی 3:2، نه 9:16. بنابراین مشکل از validation نیست؛ validation درست خطا می‌دهد. مشکل این است که درخواست AI نسبت تصویر را به‌صورت پارامتر واقعی تولید تصویر ارسال نمی‌کند.

برنامه اجرا:

1. اصلاح backend تابع `image-reframe`
   - مدل را از مسیر فعلی `google/gemini-2.5-flash-image` به مدل تصویر جدیدتر و مناسب‌تر برای edit/reframe یعنی `google/gemini-3.1-flash-image-preview` تغییر می‌دهم.
   - در payload علاوه بر prompt، پارامتر صریح خروجی تصویر اضافه می‌کنم:
     - `responseFormat.image.aspectRatio = "9:16" | "1:1" | "16:9"`
     - `responseFormat.image.imageSize = "1K"`
   - تصویر مرجعِ نسبت تصویر را حذف یا فقط به‌عنوان fallback نگه می‌دارم، چون پارامتر رسمی نسبت تصویر باید منبع اصلی کنترل ابعاد باشد.

2. سخت‌تر کردن دستور reframe بدون letterbox
   - prompt را دقیق‌تر می‌کنم که خروجی باید canvas واقعی با نسبت هدف باشد.
   - برای تصویر تبلیغاتی/متن‌دار تأکید می‌کنم کل طرح و متن اصلی حفظ شود و فضای اضافه با پس‌زمینه/بافت/المان‌های سازگار outpaint شود، نه اینکه عکس وسط قاب سیاه قرار بگیرد.

3. مدیریت retry هدفمند
   - اگر تلاش اول نسبت درست نداد، retry دوم با همان `responseFormat` و prompt سخت‌گیرانه‌تر انجام می‌شود.
   - اگر gateway پارامتر جدید را رد کند، خطای واقعی gateway در لاگ ثبت می‌شود تا به جای پیام مبهم، علت مشخص باشد.

4. حفظ validation خروجی
   - parser ابعاد PNG/JPEG/WebP را نگه می‌دارم.
   - خروجی فقط وقتی ذخیره می‌شود که نسبت واقعی فایل با نسبت خواسته‌شده سازگار باشد.
   - خطای فعلی دیگر نباید برای خروجی قابل‌اصلاح رخ دهد؛ اگر مدل واقعاً ناتوان باشد، پیام واضح‌تر برمی‌گردد.

5. استقرار و تست واقعی
   - فقط تابع `image-reframe` را deploy می‌کنم.
   - لاگ deploy را بررسی می‌کنم تا خطای bundle/import نداشته باشد.
   - سپس با درخواست مستقیم به تابع تست می‌کنم که مسیر backend اجرا شود و دیگر `1248x832` برای 9:16 برنگردد.

ریسک و کنترل:
- تغییر محدود به backend همین تابع است؛ UI و جریان آپلود دست‌نخورده می‌ماند.
- هیچ secret یا فایل auto-generated تغییر نمی‌کند.
- هزینه AI کمی بیشتر می‌شود چون مدل جدیدتر image-edit استفاده می‌شود، اما برای رسیدن به نسبت واقعی قابل‌اعتمادتر است.
