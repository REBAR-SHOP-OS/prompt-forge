## هدف نهایی
کارت‌های فیلم در ستون Pending / Library / Storage هنگام لود یا به‌روزرسانی وضعیت، دیگر بین placeholder خاکستری/مشکی و ویدئو سوییچ نکنند و تصویر کارت پایدار بماند.

## تشخیص فعلی
- کارت‌ها از `PlayableVideo` استفاده می‌کنند.
- `PlayableVideo` تا وقتی URL ویدئو resolve شود، placeholder نشان می‌دهد؛ اگر poster وجود نداشته باشد همان سطح خاکستری دیده می‌شود.
- در `onLoadedMetadata` کارت‌ها `currentTime` را تغییر می‌دهند تا فریم preview بسازند؛ همین seek اولیه می‌تواند باعث چشمک/فیلیکر شود، مخصوصاً وقتی poster نداریم یا کارت با poll دوباره render می‌شود.
- `preload="auto"` روی تعداد زیادی کارت باعث فشار لود هم‌زمان و احتمال repaint/seek بیشتر می‌شود.

## برنامه اصلاح
1. **پایدار کردن رندر داخل `PlayableVideo`**
   - برای کارت‌های thumbnail، ویدئو تا قبل از `loadeddata/canplay` روی صفحه جایگزین placeholder نشود.
   - اگر `poster` وجود دارد، poster تا آماده شدن اولین فریم روی ویدئو باقی بماند.
   - هنگام retry یا reload، به‌جای خالی/خاکستری شدن کارت، آخرین حالت پایدار یا poster حفظ شود.

2. **حذف seek اجباری از کارت‌های کوچک**
   - در کارت‌های Pending / Library / Storage، `onLoadedMetadata` که `currentTime` را به ۴ ثانیه می‌برد حذف یا محدود می‌شود.
   - نتیجه: کارت‌ها دیگر با هر metadata load یک seek و repaint محسوس انجام نمی‌دهند.
   - اگر thumbnail موجود باشد همان poster نمایش داده می‌شود؛ اگر نباشد اولین فریم طبیعی ویدئو بدون پرش نمایش داده می‌شود.

3. **کم کردن فشار لود هم‌زمان**
   - برای کارت‌های کوچک `preload` از `auto` به `metadata` تغییر می‌کند.
   - player اصلی/preview بزرگ بدون تغییر می‌ماند تا تجربه پخش اصلی خراب نشود.

4. **اعتبارسنجی**
   - بررسی می‌کنم کارت‌های مشخص‌شده در اسکرین‌شات دیگر هنگام باز شدن/لود شدن gray flash نداشته باشند.
   - تغییر فقط در UI و رفتار لود کارت‌هاست؛ منطق ساخت فیلم، آرشیو، دانلود، Storage و API تغییر نمی‌کند.
