# آیکون Regenerate + نریشن تبلیغاتی بدون اعداد

سه تغییر روی قابلیت «Product narration»:

1. افزودن آیکون **Regenerate** کنار دکمه‌ی «Product narration» برای تولید دوباره‌ی نریشن.
2. نریشن همیشه کاملاً تبلیغاتی و مناسب گفتار (voiceover) باشد.
3. فقط نام محصول استفاده شود و از آوردن عدد/کد محصول (مثل ۰۰۸) جلوگیری شود.

## تغییرات

### ۱) `supabase/functions/ad-narration/index.ts`
- **پاکسازی نام محصول قبل از ارسال به مدل:** حذف اعداد و کدها و آندرلاین‌ها از `productName`؛ مثلا `rebar_stirrup_008` → `rebar stirrup`. (regex برای حذف ارقام و کاراکترهای `_/-` و فاصله‌های اضافی، سپس trim).
- **تقویت پرامپت:** تأکید بر اینکه:
  - خروجی همیشه لحن تبلیغاتیِ پرشور و حرفه‌ای داشته باشد و کاملاً برای خوانده‌شدن به‌صورت voiceover مناسب باشد.
  - **هیچ عدد، رقم، کد محصول یا SKU** نوشته نشود (نه به‌صورت رقم و نه به‌صورت حروف مثل «double oh eight»)؛ فقط از نام توصیفی محصول استفاده شود.
- اگر بعد از پاکسازی نام محصول خالی شد، از همان نام اصلی به‌صورت fallback استفاده شود.
- استقرار مجدد edge function و تست با curl.

### ۲) `src/modules/generator-ui/components/VoiceoverDialog.tsx`
- نگه‌داشتن آخرین محصول و مدت استفاده‌شده برای regenerate:
  - state جدید `lastNarration` شامل `{ productId, seconds }` که بعد از موفقیت تولید مقداردهی می‌شود.
- بازسازی `handleGenerateNarration` تا با پارامتر دلخواه (محصول/ثانیه) هم قابل فراخوانی باشد؛ تابع کمکی برای regenerate که از `lastNarration` استفاده می‌کند.
- افزودن **دکمه‌ی آیکونی Regenerate** (آیکون `RefreshCw` از lucide که از قبل در دسترس است) درست کنار دکمه‌ی «Product narration» (همان ناحیه‌ی دایره‌ای در تصویر):
  - فقط زمانی فعال/نمایش داده شود که قبلاً یک نریشن تولید شده باشد (`lastNarration` موجود باشد).
  - کلیک → تولید دوباره با همان محصول و مدت قبلی؛ هنگام تولید، آیکون به حالت لودینگ (چرخش) برود و غیرفعال شود.
  - دارای `title`/`aria-label` مناسب («Regenerate narration»).

## بدون تغییر
- منطق صدا (TTS)، انتخاب صدا، و سایر بخش‌های دیالوگ دست‌نخورده می‌ماند.
